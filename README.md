Magma is a minimal set of single-header modules for making games and apps in C/C++, inspired by libraries like [sokol](https://github.com/floooh/sokol) and [stb](https://github.com/nothings/stb).

## Headers

| Header                | Description                                             |
| --------------------- | ------------------------------------------------------- |
| [`mgapp.h`](#mgapph)   | Platform layer — windowing, input, event loop          |
| [`mgfx.h`](#mgfxh)     | Render API abstraction — D3D11, OpenGL, OpenGL ES, Vulkan |
| [`mgmath.h`](#mgmathh) | Math — vectors, matrices, quaternions                  |

---

## mgapp.h

## mgfx.h

A render API abstraction supporting **DirectX 11**, **OpenGL**, **OpenGL ES 3**, **Vulkan**, and **Metal** (coming soon). Switch backends with a single `#define` before the include or in mgfx_init() during runtime.

```c
#define MG_IMPL
#include "mgapp.h"

#define MGFX_D3D11   // or MGFX_OPENGL, MGFX_VULKAN, MGFX_GLES3
#include "mgfx.h"

#include "triangle.glsl.h"  // generated by magma-shdc

MG_DEFINE_APP("Triangle")

static mgfx_pipeline pip;
static mgfx_buffer vb;

void mg_app_on_start(void)
{
    mgfx_init(&(mgfx_init_info){
        .handle = mg_app_handle(),
        .width  = mg_app_width(),
        .height = mg_app_height(),
        .vsync  = true
    });

    vb = mgfx_create_buffer(&(mgfx_buffer_create_info){
        .usage = MGFX_BUFFER_USAGE_VERTEX,
        .data  = (float[]){
             0.0f, -0.5f,  1.0f, 0.5f, 0.5f,
             0.5f,  0.5f,  0.5f, 1.0f, 0.5f,
            -0.5f,  0.5f,  0.5f, 0.5f, 1.0f,
             0.0f, -0.5f,  1.0f, 0.5f, 0.5f,
        },
        .size = 20 * sizeof(float)
    });

    pip = mgfx_create_pipeline(&(mgfx_pipeline_create_info){
        .vertex_attributes  = { MGFX_VERTEX_FORMAT_FLOAT2, MGFX_VERTEX_FORMAT_FLOAT3 },
        .primitive_topology = MGFX_PRIMITIVE_TOPOLOGY_LINE_STRIP,
        .shader             = get_triangle_shader(mgfx_get_shader_lang()),
    });
}

void mg_app_on_end(void)
{
    mgfx_destroy_buffer(vb);
    mgfx_destroy_pipeline(pip);
    mgfx_shutdown();
}

void mg_app_on_update(void)
{
    mgfx_begin();
    mgfx_bind_render_pass(0, (mgfx_color){ 0.05f, 0.05f, 0.05f, 1.0f });
    mgfx_bind_pipeline(pip);
    mgfx_bind_vertex_buffer(vb);
    mgfx_draw(4, 0);
    mgfx_end();
}

void mg_app_on_event(const mg_app_event *event)
{
    if (event->type == MG_APP_EVENT_RESIZE)
        mgfx_resize(event->window_width, event->window_height);
}
```

## mgmath.h

Vector, matrix and quaternion math. All functions are `static inline` — no translation unit needed.

```c
#include "mgmath.h"

// Vectors
mg_vec3 a = {1.0f, 0.0f, 0.0f};
mg_vec3 b = {0.0f, 1.0f, 0.0f};
mg_vec3 c = mg_vec3_cross(a, b);

// Quaternions
mg_vec4 q = mg_quat_from_euler(0.0f, MG_DEG2RAD * 45.0f, 0.0f);

// Matrices
mg_mat4 proj = mg_mat4_perspective(MG_DEG2RAD * 60.0f, 16.0f/9.0f, 0.1f, 1000.0f);
mg_mat4 view = mg_mat4_look_at((mg_vec3){0,2,5}, (mg_vec3){0,0,0}, (mg_vec3){0,1,0});
mg_mat4 mvp  = mg_mat4_mul(proj, view);
```

## Magma Shader Compiler (magma-shdc)

Magma ships its own shader cross-compiler built on [SPIRV-Cross](https://github.com/KhronosGroup/SPIRV-Cross) — similar in spirit to [sokol-shdc](https://github.com/floooh/sokol-tools). Write one GLSL source file with `@stage` annotations and compile it to a C header containing shader code for all target backends.

**Supported output languages:** `glsl`, `glsles`, `hlsl`, `msl`, `spirv`

### Writing a shader

```glsl
@stage vertex

in vec2 pos;
in vec3 color;
out vec3 frag_color;

void main()
{
    gl_Position = vec4(pos, 0.0, 1.0);
    frag_color = color;
}

@stage fragment

in vec3 frag_color;
out vec4 out_color;

void main()
{
    out_color = vec4(frag_color, 1.0);
}
```

### Compiling

```bash
magma-shdc --input triangle.glsl --output triangle.glsl.h --name triangle --lang hlsl,glsl,spirv
```

This produces `triangle.glsl.h` which you `#include` directly. At runtime, pass the correct variant to your pipeline using `mgfx_get_shader_lang()`:

```c
.shader = get_triangle_shader(mgfx_get_shader_lang()),
```

## Building magma-shdc

### Windows

Requires [Visual Studio 2022](https://visualstudio.microsoft.com/vs/) with MSVC and the Windows SDK.

```bat
premake5 vs2022
MSBuild Magma.sln
```

### Linux

> Tested on Arch Linux. Wayland users should use Xwayland.

```bash
# Dependencies
sudo pacman -Sy base-devel premake vulkan-devel libegl xorg-server
yay -S spirv-cross   # or: paru -S spirv-cross

# Generate & build
premake5 gmake
make
```
